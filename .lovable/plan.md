

# Add User Authentication with Supabase

## Overview
Add email-based authentication (sign up, log in, stay logged in) with user profiles (name, optional profile picture) using Lovable Cloud / Supabase.

## Prerequisites
- Enable Lovable Cloud (Supabase) on the project — this must be done before any database or auth work.

## Database

**Profiles table** (via migration):
```sql
CREATE TABLE public.profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  name TEXT NOT NULL DEFAULT '',
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can read own profile" ON public.profiles
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "Users can update own profile" ON public.profiles
  FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "Users can insert own profile" ON public.profiles
  FOR INSERT WITH CHECK (auth.uid() = id);
```

**Trigger** to auto-create profile on signup:
```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger AS $$
BEGIN
  INSERT INTO public.profiles (id, name)
  VALUES (NEW.id, COALESCE(NEW.raw_user_meta_data->>'name', ''));
  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

**Avatar storage bucket**:
```sql
INSERT INTO storage.buckets (id, name, public) VALUES ('avatars', 'avatars', true);
```
With RLS policies allowing users to upload/manage their own files.

## New Files

1. **`src/integrations/supabase/client.ts`** — Supabase client init (auto-generated by Lovable Cloud)
2. **`src/contexts/AuthContext.tsx`** — Auth context providing user/session state, sign up, log in, log out functions. Uses `onAuthStateChange` listener set up before `getSession()`.
3. **`src/pages/Auth.tsx`** — Combined login/signup page with toggle. Fields: email, password, name (signup only). Styled to match the dark warm theme with coral accents.
4. **`src/pages/Profile.tsx`** — Simple profile page showing name, avatar, with ability to edit name and upload avatar photo.
5. **`src/components/ProtectedRoute.tsx`** — Wrapper that redirects to `/auth` if not logged in.

## Modified Files

1. **`src/App.tsx`** — Wrap with `AuthProvider`, add routes for `/auth` and `/profile`, protect `/` with `ProtectedRoute`.
2. **`src/pages/Index.tsx`** — Replace the placeholder avatar in header with real user avatar/name and a dropdown with "Profile" and "Log out" options.

## Auth Flow
- Sign up: email + password + name → creates auth user → trigger creates profile
- Log in: email + password
- Session persistence: Supabase handles via `onAuthStateChange`
- The auth page matches the app's dark theme with coral buttons

## Design Decisions
- Auth page: centered card on dark background, "Roamora" wordmark at top, toggle between Login/Sign Up
- Profile page: simple card with avatar upload circle, name field, save button
- Header avatar becomes clickable with dropdown menu when logged in
